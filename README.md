# Сетевой чат (TCP, WPF, многопоточность)

Учебный проект многопользовательского чата на C# с графическим интерфейсом на WPF. Сервер принимает подключения по TCP и обслуживает клиентов в отдельных потоках. Клиент позволяет отправлять сообщения в общий чат, видеть список участников и системные уведомления.

## Структура решения

- **ChatServer** – серверное приложение (WPF)
  - `ChatServerCore` – основная логика: запуск TcpListener, приём подключений, хранение списка клиентов, broadcast
  - `ClientHandler` – обработчик одного клиентского соединения (поток на каждого)
  - `MainWindow` – окно сервера: лог событий, счётчик клиентов, кнопки управления
- **ChatClient** – клиентское приложение (WPF)
  - `ChatClientCore` – сетевая логика: TcpClient, подключение, отправка/приём сообщений
  - `MainWindow` – окно клиента: поле ввода, лента сообщений, список пользователей онлайн

### Основные технические детали

- Язык: C# (совместимость с C# 7.3 и выше)
- Платформа: .NET 8.0 (Windows)
- Интерфейс клиента: WPF
- Интерфейс сервера: WPF (окно с логом и управлением)
- Сеть: TCP, классы `TcpListener`, `TcpClient`, `NetworkStream`
- Многопоточность: выделенный поток на каждое клиентское соединение, синхронизация через `lock`
- Обновление UI из фоновых потоков: `Dispatcher.Invoke`
- Событийная модель на сервере: явные события `OnClientConnected`, `OnClientDisconnected`, `OnMessageReceived`

## Требования к сборке и запуску

- Windows 7/10/11
- .NET 8.0 SDK
- Visual Studio 2022 (или Rider с поддержкой WPF)

## Сборка и первый запуск

1. Откройте `ChatSolution.sln` в Visual Studio.
2. Убедитесь, что для проектов `ChatServer` и `ChatClient` указана целевая платформа `.NET 8.0`.
3. Соберите решение (`Build → Build Solution`).

### Запуск из Visual Studio (множественный запуск)

1. В обозревателе решений щёлкните правой кнопкой мыши по узлу решения и выберите **Свойства**.
2. В разделе **Запускаемый проект** выберите опцию **Несколько запускаемых проектов**.
3. Для `ChatServer` и `ChatClient` установите действие **Запуск**.
4. Нажмите `F5` – откроются два окна (сервер и первый клиент).

### Запуск исполняемых файлов

Скомпилируйте решение и запустите из папок сборки:

- `ChatServer\bin\Debug\net8.0-windows\ChatServer.exe`
- `ChatClient\bin\Debug\net8.0-windows\ChatClient.exe`

Для тестирования с несколькими клиентами можно запустить `ChatClient.exe` повторно.

## Тестирование

1. На сервере нажмите **Запустить** (порт по умолчанию 5000).
2. В клиенте заполните поля:
   - **IP:** `127.0.0.1`
   - **Порт:** `5000`
   - **Никнейм:** уникальное имя (например, `User1`)
3. Нажмите **Подключиться**.
4. Запустите второй клиент и подключитесь с другим ником (`User2`).
5. Обменивайтесь сообщениями – каждый видит сообщения другого, а также свои собственные (локально).
6. В правой панели отображается актуальный список пользователей.
7. Системные сообщения о входе/выходе участников выводятся в ленте.
8. Проверьте:
   - Отправку пустой строки (игнорируется)
   - Очень длинное сообщение (обрезается до 500 символов)
   - Закрытие клиента без нажатия «Отключиться» (остальные получают уведомление о выходе)

## Протокол обмена (прикладной уровень)

Все сообщения передаются в текстовом виде, оканчиваются переводом строки.

| Команда / Формат                | Назначение |
|---------------------------------|------------|
| `/join <никнейм>`               | Запрос подключения с указанием псевдонима |
| `/list`                         | Запрос списка активных пользователей |
| `OK: <текст>`                   | Успешная авторизация |
| `ERROR: <текст>`                | Ошибка (ник занят, неверный формат и т.п.) |
| `SERVER: <текст>`               | Системное уведомление всем клиентам |
| `USERLIST:<ник1>,<ник2>,...`   | Передача списка пользователей |
| `<отправитель>: <текст>`        | Обычное сообщение в чат (рассылается сервером всем, кроме отправителя) |

## Примечания по реализации

- Максимальная длина одного сообщения – 500 символов (более длинные обрезаются).
- Сервер хранит список клиентов в словаре `(никнейм → StreamWriter)`, что позволяет быстро исключать отправителя из рассылки.
- При разрыве соединения клиент автоматически удаляется из списка, а остальные получают уведомление.
- Клиент сразу отображает собственное сообщение в ленте, не ожидая эха от сервера.
- Все обновления пользовательского интерфейса из фоновых потоков выполняются через `Dispatcher.Invoke`.

Проект выполнен в рамках лабораторной работы по системному программному обеспечению.
